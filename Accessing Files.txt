
3.Accessing Files

Attributes −
•	Owner permissions −Actions the owner of the file can perform on the file determined by owner's permissions.
•	Group permissions − Actions for user of the group that a file belongs to that can perform on the file is determinerd by the group's permissions.
•	Other (world) permissions − Action all other users can perform on the file determined by permissions for others.

File Access Modes-
-The permissions of a file are the basic building blocks of Unix permissions are the read, write, and execute permissions 
-Denote as: read (r), write (w), execute (x) .

1.Read: Grants the capability to read, i.e., view the contents of the file.
2.Write:Grants the capability to modify, or remove the content of the file.
3.Execute:User with execute permissions can run a file as a program.

Execute permission:	--x
Write permission :	-w-
Execute and write permission: 1 (execute) + 2 (write) = 3 :	-wx
Read permission:	r--
Read and execute permission: 4 (read) + 1 (execute) = 5 :	r-x
Read and write permission: 4 (read) + 2 (write) = 6	:       rw-
All permissions: 4 (read) + 2 (write) + 1 (execute) = 7	:       rwx

Two commands are available to change the owner and the group of files −
-	chown :The chown command stands for "change owner" and is used to change the owner of a file.
-	chgrp : The chgrp command stands for "change group" and is used to change the group of a file.
-	chmod :To change the file or the directory permissions, you use the change mode command.
